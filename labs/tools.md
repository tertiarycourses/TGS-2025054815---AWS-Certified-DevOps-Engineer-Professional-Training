# Tools Reference

## Required Tools

- AWS account or trainer-provided sandbox account
- Web browser
- Notes application or markdown editor

## Recommended AWS Tools and Services

- IAM
- AWS Organizations
- AWS CloudTrail
- AWS Config
- AWS CloudFormation
- AWS CDK
- AWS CodeCommit or external Git provider
- AWS CodeBuild
- AWS CodeDeploy
- AWS CodePipeline
- Amazon S3 artifact buckets
- AWS Lambda
- Amazon ECS and Amazon ECR
- Elastic Beanstalk
- AWS Systems Manager
- AWS Parameter Store
- AWS Secrets Manager
- Amazon CloudWatch
- AWS X-Ray
- Amazon EventBridge
- AWS Backup
- Route 53
- Auto Scaling
- AWS Fault Injection Service
- AWS Security Hub
- Amazon GuardDuty
- AWS Trusted Advisor
- AWS Budgets and Cost Explorer

## Cost Safety

- Use trainer-provided accounts where available.
- Avoid long-running compute, build projects, container services, or provisioned databases unless explicitly assigned.
- Delete test stacks, pipelines, functions, queues, clusters, alarms, and buckets after labs if instructed.
- Use small sample applications and short build runs.
- Set budget alerts before using hands-on services.
- Check Billing and Cost Management after labs.

## Learner Tips

- Think in systems: source, build, deploy, observe, remediate, govern.
- Know the difference between immutable, rolling, blue/green, canary, and linear deployments.
- Treat monitoring as part of the release pipeline.
- Practice explaining rollback, remediation, and least-privilege decisions.
- Keep a weak-topic list for exam revision.
